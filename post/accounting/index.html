记账的演变之旅

2023-03-05 Views life

不知不觉已经保持了很多年的记账习惯，中间做过一些调整，不断在追问和尝试中寻找每一个阶段更合适自己的记账方法

高中时候 CC 有记账的习惯，还给我看过他的记账本，并且讲了一下他大概记些什么东西。彼时我没有收入，能花的钱也不多，所以一直没实操过。

大一的时候参加了书院里一场讲大数据的小型讲座，第一次接触大数据这个东西，说实话非常震撼，讲座之后我觉得可以实践起来，于是从收集自己产生的“小数据”开始，在 Excel 里记录自己的一些生活细节，比如起床入睡时间、洗头、洗澡、刷牙等等日常行为。那时候我洗澡用肥皂，统计一段时间下来之后，发现一块肥皂可以使用 20-22 次(可能不是这个数据，等我有时间核查一下，不过，长时间的数据统计结论是一个偏差范围在 1-2 的数)。

大概在同一时期(回头有时间也查一下确定时期)，我也在纸质本子上记录日常支出(没有收入)，由于一日三餐主要是吃食堂，所以隔一段时间需要去自助机器上查自己校园卡的消费记录，拍好照回寝室记在本子上，现在翻到以前的相册还能看到这些照片。

大一的时候我在南科公益负责记账，也会需要统计一些项目的支出，不过没啥复杂的项目。相反每次聚餐 AA 的账单细节倒是更多，大家一起吃晚饭各自回寝室之后我再加会班做个表格出来，把每个人 A 的钱私聊发出去，在这个过程中摸索使用 Excel 的公式计算功能。

纸质记账的习惯一直延续到毕业之后，有时这种记录也承担了一部分日记的功能。纸质的缺点是不好统计，不过早期也没什么统计需求，能记能查就行。中间偶尔会做一些调整，比如不同日子占的行数不一样，没法跳过中间的日子直接记后面的日子，索性改成了每一天统一预留固定的行数，如果留多了，看起来也美观。诸如此类，时时调整。

毕业之后自己租房，提前把工资都花出去了，每个月要算好还款时间，所以在 Excel 上开辟了一个资产记录的表。纸质记账也在同步进行，更加详细。后来慢慢发展成 Excel 上记所有账户每一天的余额变动和统计，纸质本子上记录收支细节。这个模式又持续了好几年。

随着 Excel 里的越记越多，几次突破了默认的列数上限，加之统计起来也不太方便，慢慢想着寻找更加成熟的专门工具来记账。在间歇性地试用了一些工具后，这个想法发展成了自己写一个记账工具。一来自己的技术栈离得不算远，趁着造轮子的机会可以玩一些新的技术；二来自己写有很大的自由度，可以完全贴合自己的需求。

于是我开始做项目计划、简易架构、技术选型等等。在设计数据结构的阶段，我仔细的审视了自己的需求，用 TS 写了一些 model，这个时候我发现，好像我的需求用 notion database 可以完全满足。加上自己的造轮子计划也已经在时间上拉的很长了，我开始认证考虑 notion 的选项。

经过不太长的思考，我放弃了自己造轮子的计划，转而在 notion 里面按照自己设计的数据结构创建对应的 database，然后利用 notion api 连接各个交互入口。这个方案的好处在于，我几乎立刻就拥有了现成的前端和后端。日常记账使用 iOS 的快捷指令调用 notion api 插入结构化的数据，基本上满足了我原本的需求。

唯一的不足是 notion database 的统计能力有限，有一些统计维度不支持。于是我自己写了一个网页，通过 notion api 拿到全部数据，自己实现想要的统计。

最近伴侣也慢慢开始采用我的这一套记账方案，我把自己实践出来的 notion template 分享出来，教她如何使用。
